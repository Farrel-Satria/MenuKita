1. GAMBARAN UMUM PROJEK

MenuKita adalah aplikasi Android untuk manajemen menu restoran yang terhubung ke Firebase Realtime Database. Aplikasi ini memiliki dua jenis pengguna:

RoleKemampuanDashboard
AdminMenambah, Mengedit, Menghapus, dan Melihat semua menuMainActivity (List View)
User (Pelanggan)Melihat daftar menu, Search, dan Filter kategoriUserDashboardActivity (Grid View)
📌 Poin Penting: Database menggunakan Firebase Realtime Database yang bersifat realtime — ketika admin menambah/edit/hapus menu, perubahan langsung terlihat tanpa perlu refresh.

4. REPOSITORY (Koneksi ke Firebase)

📄 repository/MenuRepository.kt

Kelas yang menangani semua operasi CRUD (Create, Read, Update, Delete) menu ke Firebase Realtime Database. Fungsi addMenu() menggunakan push().key untuk generate ID unik. Fungsi getMenus() menggunakan addValueEventListener yang bersifat realtime — setiap perubahan data di Firebase akan langsung memanggil callback. Fungsi updateMenu() menimpa data berdasarkan ID, dan deleteMenu() menghapus node berdasarkan ID.

Source Code:

package com.example.menukita.repository import com.example.menukita.model.Menu import com.google.firebase.database.DataSnapshot import com.google.firebase.database.DatabaseError import com.google.firebase.database.FirebaseDatabase import com.google.firebase.database.ValueEventListener class MenuRepository { private val database = FirebaseDatabase.getInstance("https://menukita-feb11-default-rtdb.asia-southeast1.firebasedatabase.app") private val menuRef = database.getReference("menus") // CREATE fun addMenu(menu: Menu, onComplete: (Boolean) -> Unit) { val id = menuRef.push().key ?: return val newMenu = menu.copy(id = id) menuRef.child(id).setValue(newMenu) .addOnCompleteListener { task -> onComplete(task.isSuccessful) } } // READ (Realtime Listener) fun getMenus(onDataChange: (List<Menu>) -> Unit, onError: (String) -> Unit) { menuRef.addValueEventListener(object : ValueEventListener { override fun onDataChange(snapshot: DataSnapshot) { val menuList = mutableListOf<Menu>() for (menuSnapshot in snapshot.children) { try { val menu = menuSnapshot.getValue(Menu::class.java) menu?.let { menuList.add(it) } } catch (e: Exception) { // Log error to identify problematic data in Firebase Console android.util.Log.e("MenuRepository", "Error parsing menu at key: ${menuSnapshot.key}", e) } } onDataChange(menuList) } override fun onCancelled(error: DatabaseError) { onError(error.message) } }) } // UPDATE fun updateMenu(menu: Menu, onComplete: (Boolean) -> Unit) { val id = menu.id ?: return menuRef.child(id).setValue(menu) .addOnCompleteListener { task -> onComplete(task.isSuccessful) } } // DELETE fun deleteMenu(id: String, onComplete: (Boolean) -> Unit) { menuRef.child(id).removeValue() .addOnCompleteListener { task -> onComplete(task.isSuccessful) } } }

📄 repository/UserRepository.kt

Kelas yang menangani autentikasi user. Fungsi register() mengecek duplikasi email terlebih dahulu menggunakan orderByChild("email").equalTo(email), lalu menyimpan user baru dengan key berupa email yang dimodifikasi (titik dan @ diganti underscore). Fungsi login() mencari user berdasarkan email dan mengembalikan objek User jika ditemukan.

package com.example.menukita.repository import android.util.Log import com.example.menukita.model.User import com.google.firebase.database.DataSnapshot import com.google.firebase.database.DatabaseError import com.google.firebase.database.FirebaseDatabase import com.google.firebase.database.ValueEventListener class UserRepository { private val db = FirebaseDatabase.getInstance("https://menukita-feb11-default-rtdb.asia-southeast1.firebasedatabase.app") private val userRef = db.getReference("users") // REGISTER - simpan user baru ke Firebase fun register(user: User, onComplete: (Boolean, String?) -> Unit) { val email = user.email if (email.isNullOrEmpty()) { onComplete(false, "Email tidak boleh kosong") return } // Cek apakah email sudah terdaftar (cari by field email, bukan key) userRef.orderByChild("email").equalTo(email) .addListenerForSingleValueEvent(object : ValueEventListener { override fun onDataChange(snapshot: DataSnapshot) { if (snapshot.exists()) { onComplete(false, "Email sudah terdaftar!") } else { // Key bisa apa saja, kita pakai email yang sudah dimodifikasi val emailKey = email.replace(".", "_").replace("@", "_at_") Log.d("UserRepo", "Menyimpan user dengan key: $emailKey") userRef.child(emailKey).setValue(user).addOnCompleteListener { task -> if (task.isSuccessful) { Log.d("UserRepo", "Register BERHASIL!") onComplete(true, null) } else { val err = task.exception?.message ?: "Gagal menyimpan" Log.e("UserRepo", "Register GAGAL: $err") onComplete(false, err) } } } } override fun onCancelled(error: DatabaseError) { onComplete(false, error.message) } }) } // LOGIN - cari user berdasarkan field email (key boleh apa saja: "admin", "user1", dll.) fun login(email: String, onComplete: (User?, String?) -> Unit) { Log.d("UserRepo", "Login dengan email: $email") userRef.orderByChild("email").equalTo(email) .addListenerForSingleValueEvent(object : ValueEventListener { override fun onDataChange(snapshot: DataSnapshot) { if (snapshot.exists()) { Log.d("UserRepo", "User ditemukan!") val userSnapshot = snapshot.children.first() val user = userSnapshot.getValue(User::class.java) onComplete(user, null) } else { Log.w("UserRepo", "User tidak ditemukan untuk email: $email") onComplete(null, "Akun tidak ditemukan. Silakan daftar terlebih dahulu.") } } override fun onCancelled(error: DatabaseError) { Log.e("UserRepo", "Login error: ${error.message}") onComplete(null, error.message) } }) } }

package com.example.menukita.util import android.content.Context import android.net.ConnectivityManager import android.net.NetworkCapabilities object NetworkUtils { fun isNetworkAvailable(context: Context): Boolean { val connectivityManager = context.getSystemService(Context.CONNECTIVITY_SERVICE) as ConnectivityManager val network = connectivityManager.activeNetwork ?: return false val activeNetwork = connectivityManager.getNetworkCapabilities(network) ?: return false return when { activeNetwork.hasTransport(NetworkCapabilities.TRANSPORT_WIFI) -> true activeNetwork.hasTransport(NetworkCapabilities.TRANSPORT_CELLULAR) -> true activeNetwork.hasTransport(NetworkCapabilities.TRANSPORT_ETHERNET) -> true else -> false } } }

12. PERTANYAAN YANG MUNGKIN DITANYA DOSEN

PertanyaanJawaban
Kenapa pakai Firebase Realtime Database?Karena mendukung sinkronisasi data secara realtime dan mudah diintegrasikan ke Android tanpa perlu membuat backend sendiri.
Apa perbedaan addValueEventListener vs addListenerForSingleValueEvent?addValueEventListener terus mendengar perubahan data (realtime), sedangkan addListenerForSingleValueEvent hanya membaca data sekali saja.
Kenapa pakai View Binding?Lebih aman (null-safe, type-safe), lebih efisien daripada findViewById, dan kode lebih bersih.
Bagaimana role-based access bekerja?Saat login, sistem mengecek field "role" dari Firebase. Admin diarahkan ke MainActivity (CRUD), User ke UserDashboardActivity (view only).
Apa fungsi Glide?Library untuk load gambar dari URL ke ImageView secara efisien, termasuk fitur caching, placeholder, dan error handling.
Kenapa RecyclerView, bukan ListView?RecyclerView lebih efisien karena me-recycle (daur ulang) view yang tidak terlihat di layar, menghemat memory.
Apa itu data class di Kotlin?Kelas khusus yang otomatis generate toString(), equals(), hashCode(), copy(). Cocok untuk model data.
Kenapa field di data class nullable (?)?Karena data dari Firebase bisa kosong/null. Dengan nullable + default value, Firebase bisa membuat objek tanpa crash.
